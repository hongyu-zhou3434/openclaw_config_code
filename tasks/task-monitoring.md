# 任务监控配置

## 监控维度

### 1. 时间监控
- **执行时长**: 超过预估时间 2x 触发警告
- **超时阈值**: 默认 300 秒（5分钟）
- **检查间隔**: 每 30 秒检查一次

### 2. 资源监控
- **CPU 阈值**: > 90% 触发警告
- **内存阈值**: > 85% 触发警告
- **磁盘空间**: < 10% 剩余空间触发警告

### 3. 输出监控
- **stderr 输出**: 持续输出错误信息触发警告
- **退出码**: 非 0 退出码标记为失败
- **日志大小**: 超过 100MB 触发警告

### 4. 进度监控
- **无更新阈值**: > 5 分钟无输出触发检查
- **进度标记**: 任务应定期报告进度

### 5. 状态监控
- **zombie 进程**: 自动清理
- **failed 状态**: 记录并通知
- **timeout 状态**: 自动终止并记录

### 6. 结果监控
- **文件缺失**: 检查预期输出文件
- **格式错误**: 验证输出格式
- **内容验证**: 检查关键内容存在

## 监控命令

```bash
# 检查系统资源
openclaw status

# 查看运行中的任务
ps aux | grep -E "(openclaw|python|node)"

# 查看任务日志
tail -f logs/task-*.log
```

## 告警方式

1. **日志记录**: 所有告警写入 `.learnings/ERRORS.md`
2. **self-improving-agent**: 自动分析并记录
3. **控制台输出**: 实时显示警告信息
